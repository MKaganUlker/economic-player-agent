# Economic Player Agent

> An autonomous AI agent that lives inside an economic simulation, makes long-term decisions, spends resources, learns from experience, and optimizes its financial future.

---

## Overview

Most AI agents only generate answers.

**Economic Player Agent takes actions.**

This project creates an autonomous economic player controlled by a local LLM.

The agent lives inside a simulated economy and must maximize:

> **Long-term net worth**

Every simulation cycle:

1. Observe the current world state.
2. Reason about possible actions.
3. Select an action.
4. Execute the action through tools.
5. Observe the result.
6. Update memory.
7. Continue improving.

The environment is deterministic and reproducible, while the decision-making process is generated by a local LLM.

---

# Core Concept

Traditional simulations rely on hard-coded rules:

```python
if inflation_high:
    invest()

if cash_low:
    save()
````

This project replaces fixed decision logic with an autonomous reasoning loop:

```
Environment
     |
     v
State Observation
     |
     v
LLM Reasoning
     |
     v
Action Selection
     |
     v
Tool Execution
     |
     v
Updated Environment
     |
     v
Future Decisions
```

The LLM does not control the world directly.

It controls an agent that interacts with the world through tools.

# Current Features

## AI Agent

* Local LLM powered reasoning
* ReAct-style agent architecture
* Structured JSON decisions
* Goal-oriented decision making
* Tool-based action execution

## Economic Simulation

The simulated player has:

* Cash
* Salary
* Expenses
* Investments
* Debt
* Capabilities
* Memory

The economy contains:

* Inflation
* Stock market returns
* Unemployment

## Capability System

The agent cannot instantly gain unlimited abilities.

Capabilities have:

* Cost
* Cooldown
* Historical tracking

Example:

```
Month 1

Action:
develop_capability

Capability:
machine_learning

Cost:
2000

Cooldown:
6 months
```

The agent must decide when improving itself is worth the opportunity cost.

---

## Memory System

The agent remembers previous decisions.

Example:

```json
{
    "month": 3,
    "action": "develop_capability",
    "capability": "machine_learning",
    "cost": 2000
}
```

The goal is moving from:

```
observe -> decide -> forget
```

to:

```
observe -> decide -> remember -> improve
```
